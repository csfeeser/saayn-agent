# Lab 2: Initializing the Supermarket (The Barcode Scan)

## Objective
In this lab, you will perform your first "Inventory Scan." You will take an existing Go file and use the SAAYN-Agent to wrap its logic in physical "Barcodes" (Markers). This is the fundamental step that allows the AI to understand where it is allowed to work.

- Using `saayn init` to identify logic blocks
- Inspecting the generated `CHUNK_START` and `CHUNK_END` markers
- Exploring the `chunk-registry.json` (The Store Ledger)

## Procedure

1.  Let's look at a "Raw" payments logic file before it is barcoded.
    `student@ubuntu:~/saayn-agent$` `cat internal/logic/payments.go`

2.  Initialize the SAAYN-Agent on the logic directory.
    `student@ubuntu:~/saayn-agent$` `./saayn init ./internal/logic`
    ```text
    🔍 Scanning ./internal/logic...
    📦 Found 3 logic blocks.
    🔖 Applying Barcodes...
    ✅ Initialization complete. Registry created at .saayn/chunk-registry.json
    ```

3.  Observe how the source code has been modified with physical boundaries.
    `student@ubuntu:~/saayn-agent$` `cat internal/logic/payments.go`
    ```go
    // SAAYN:CHUNK_START:pay-v1-8fb2a3
    func ProcessPayment(amount float64) { ... }
    // SAAYN:CHUNK_END:pay-v1-8fb2a3
    ```

4.  Inspect the "Store Ledger" to see how the agent tracks your code.
    `student@ubuntu:~/saayn-agent$` `cat .saayn/chunk-registry.json`
    ```json
    {
      "uuid": "pay-v1-8fb2a3",
      "file_path": "internal/logic/payments.go",
      "content_hash": "a1b2c3d4e5f6...",
      "language_hint": "go"
    }
    ```

## Conclusion
You have successfully "Shelved" your code. By adding these markers, you have defined the "Blast Radius" for any future AI edits. The agent now knows exactly which lines of code belong to which identity.
