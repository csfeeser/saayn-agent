# Lab 5: The Emergency Brake (Atomic Recovery)

## Objective

In this lab, you will simulate a "Catastrophic Failure." You will interrupt the SAAYN-Agent in the middle of a critical file update (the "Point of No Return"). You will then learn how to use the `recover` command to read the **Transaction Journal** and automatically restore the codebase to its original, pre-crash state.

- Simulating a process crash during the `EXECUTING` state
- Inspecting the `.saayn/journal/` directory for "inflight" transactions
- Executing `saayn recover` to roll back partial changes
- Verifying that the codebase has returned to a consistent "All Clear" status

## Procedure

1.  Confirm your environment is clean before starting the "Crash" simulation.
    `student@ubuntu:~/saayn-agent$` `./saayn verify`
    ```text
    ✅ Integrity Verified.
    ```

2.  We will now start an edit, but we will "Kill" the process manually to simulate a crash. 
    `student@ubuntu:~/saayn-agent$` `./saayn edit pay-v1-8fb2a3 --intent "Update tax logic for 2026."`

3.  Approve the plan when prompted by typing `y` and pressing **ENTER**.
    ```text
    ? Do you approve this implementation plan? [y/N] y
    ```

4.  **IMMEDIATELY** after pressing Enter, press **CTRL+C** repeatedly to kill the program before it finishes the "Rename" step.
    `^C^C^C`
    > **Note:** If you were too fast or too slow, don't worry. The goal is to leave the `.saayn/journal` file behind.

5.  Check the "Journal" directory. Even though the program stopped, the **Manager** left a record of what it was *trying* to do.
    `student@ubuntu:~/saayn-agent$` `ls .saayn/journal/`
    ```text
    op-1234.json  pay-v1-8fb2a3.tmp
    ```

6.  Try to run a normal command. The agent will detect that the "Supermarket" is in an unsafe state.
    `student@ubuntu:~/saayn-agent$` `./saayn verify`
    ```text
    ❌ SYSTEM_LOCKED: A previous transaction (op-1234) did not complete.
    ⚠️  Codebase may be in an inconsistent state. Recovery required.
    ```

7.  Execute the **Recovery** command. This tells the agent to look at the Journal, delete any orphaned `.tmp` files, and ensure the original files are unlocked.
    `student@ubuntu:~/saayn-agent$` `./saayn recover`
    ```text
    🛠️  Initiating Recovery for op-1234...
    📂 Found Journal entry for pay-v1-8fb2a3.
    ♻️  Rolling back partial changes...
    🧹 Cleaning up temporary artifacts...
    ✅ Recovery successful. The Sovereign Market is back in IDLE state.
    ```

8.  Perform a final audit to prove the "Emergency Brake" worked perfectly.
    `student@ubuntu:~/saayn-agent$` `./saayn verify`
    ```text
    🔍 Auditing the Sovereign Market...
    ✅ Integrity Verified. (Code restored to pre-crash state).
    ```
