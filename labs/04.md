# Lab 4: The Chaos Test (Detecting Manual Drift)

## Objective

In this lab, you will intentionally "break the rules" of the Sovereign Market. You will simulate a common developer mistake: modifying code manually without using the Agentic Protocol. You will then use the `verify` command to see how SAAYN-Agent acts as a "Digital Policeman" to protect the integrity of the BIG-CCC codebase.

- Intentionally introducing "Manual Drift" into a barcoded chunk
- Running the `saayn verify` audit tool
- Understanding the difference between "Expected Hash" and "Found Hash"
- Identifying "Illegal Intrusions" (code added outside of markers)

## Procedure

1.  Confirm that your codebase is currently "Clean."
    `student@ubuntu:~/saayn-agent$` `./saayn verify`
    ```text
    🔍 Auditing the Sovereign Market...
    📦 Checking Chunk: pay-v1-8fb2a3 [internal/logic/payments.go]
    ✅ Integrity Verified.
    ✨ ALL CLEAR: Codebase is synchronized with the Sovereign Registry.
    ```

2.  Now, let's "Commit a Crime." Open the payments file with a standard text editor.
    `student@ubuntu:~/saayn-agent$` `nano internal/logic/payments.go`

3.  Manually change the fee from `1.03` to `1.05` inside the `pay-v1-8fb2a3` block. 
    > **Note:** Do NOT use the `saayn edit` command. Change the text directly.

    ```go
    // SAAYN:CHUNK_START:pay-v1-8fb2a3
    func ProcessPayment(amount float64, currency string) float64 {
        if currency != "USD" {
            amount = amount * 1.05 // Manually changed from 1.03
        }
        return amount
    }
    // SAAYN:CHUNK_END:pay-v1-8fb2a3
    ```

4.  Save the file (**CTRL+O**, **ENTER**) and exit (**CTRL+X**).

5.  Run the Auditor. This is what a Senior Engineer or a CI/CD pipeline would do before a release.
    `student@ubuntu:~/saayn-agent$` `./saayn verify`

6.  Observe the failure. The agent has detected that the "Seal" is broken.
    ```text
    🔍 Auditing the Sovereign Market...
    📦 Checking Chunk: pay-v1-8fb2a3 [internal/logic/payments.go]
    ❌ DRIFT_DETECTED: Content of pay-v1-8fb2a3 has changed outside of SAAYN!
       Expected: b4c5d6e7...
       Found:    a9z8y7x6...

    ⚠️  AUDIT FAILED: 1 issues detected. The shelf is untidy.
    ```

7.  Now, let's try a different type of "Chaos." Add a new line of code **outside** the markers at the bottom of the file.
    `student@ubuntu:~/saayn-agent$` `echo "func RogueFunction() {}" >> internal/logic/payments.go`

8.  Run the Auditor again.
    `student@ubuntu:~/saayn-agent$` `./saayn verify`
    ```text
    🔍 Auditing the Sovereign Market...
    ...
    ⚠️  AUDIT FAILED: 2 issues detected.
    ```
    > **Teaching Moment:** SAAYN-Agent flags this because the "Inventory" no longer matches the "Physical Shelf." In a production system, untracked code is a security risk.

## Conclusion

You have successfully triggered the "Chaos Alarms." By attempting to bypass the Agentic Protocol, you proved that SAAYN-Agent provides **Non-Repudiation**: no change can happen without being recorded in the Registry. In the BIG-CCC ecosystem, this ensures that every line of code can be traced back to an approved Business Intent.
