# Lab 3: The Intent-Driven Edit (Directing the AI)

## Objective

In this lab, you will use the SAAYN-Agent to implement a real business requirement. You will learn that in a production environment like BIG-CCC, "Just Coding" is a failure. You must instead provide a clear **Business Intent**, review the AI's **Plan**, and then allow the **Atomic Engine** to swap the code safely.

- Providing a high-level Business Intent to the CLI
- Reviewing the AI-generated "Change Proposal"
- Executing the 16-Step Atomic Pipeline
- Verifying the "Seal" of the new code

## Procedure

1.  Identify the chunk you want to edit. We will modify the `ProcessPayment` function.
    `student@ubuntu:~/saayn-agent$` `./saayn list`
    ```text
    ID: pay-v1-8fb2a3 | Path: internal/logic/payments.go | Lang: go
    ```

2.  Run the `edit` command. We will pass a "Business Intent" that mirrors a real BIG-CCC security requirement: adding a 3% processing fee for international transactions.
    `student@ubuntu:~/saayn-agent$` `./saayn edit pay-v1-8fb2a3 --intent "Apply a 3% international fee if the currency is not USD."`

3.  The Agent will now generate a **Plan**. Review the output carefully.
    ```text
    🤖 AI PLANNER:
    Proposed Change for [pay-v1-8fb2a3]:
    - Add 'currency' string parameter to ProcessPayment.
    - Calculate 3% fee if currency != "USD".
    - Update logic to return the new total.

    ? Do you approve this implementation plan? [y/N]
    ```

4.  Type `y` and press **ENTER**. The agent will now invoke the **Coder** and the **Validator**.

5.  The **Validator** (The Bouncer) will now check the AI's work for syntax errors.
    ```text
    🔄 STATE_CHANGE: VALIDATING
    ✅ SYNTAX_CHECK: GoAdapter confirms valid Go code.
    ✅ SEAL_VERIFIED: Proposal hash matches.
    🔄 STATE_CHANGE: APPROVED
    ```

6.  The **Engine** now performs the "Atomic Rename" to ensure the file is never corrupted.
    ```text
    💾 Committing changes...
    🔄 STATE_CHANGE: EXECUTING
    ✅ File internal/logic/payments.go updated.
    🔄 STATE_CHANGE: EXECUTED
    ```

7.  Inspect the file to see the results. Notice the markers are still perfectly intact.
    `student@ubuntu:~/saayn-agent$` `cat internal/logic/payments.go`
    ```go
    // SAAYN:CHUNK_START:pay-v1-8fb2a3
    func ProcessPayment(amount float64, currency string) float64 {
        if currency != "USD" {
            amount = amount * 1.03
        }
        return amount
    }
    // SAAYN:CHUNK_END:pay-v1-8fb2a3
    ```

8.  Run a final audit to ensure the new "Barcode" is registered in the ledger.
    `student@ubuntu:~/saayn-agent$` `./saayn verify`
    ```text
    🔍 Auditing the Sovereign Market...
    📦 Checking Chunk: pay-v1-8fb2a3 [internal/logic/payments.go]
    ✅ Integrity Verified. (New Hash: b4c5d6...)
    ```

## Conclusion

You have successfully directed the AI to modify production logic. By using the `--intent` flag, you ensured that the "Why" (The Business Rationale) was captured. By reviewing the "Plan," you maintained human sovereignty. And by using the "Atomic Engine," you ensured that if the system had crashed mid-way, your code would remain uncorrupted.
