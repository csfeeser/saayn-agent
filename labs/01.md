# Lab 1: Establishing the SAAYN Workstation

## Objective

In this lab, you will prepare a fresh Ubuntu 24.04 environment for building and running the SAAYN-Agent. You will install a modern Go toolchain directly from the official source, configure your environment, and build the SAAYN CLI from source.

* Installing a current version of Go (matching project requirements)
* Configuring the Linux PATH for Go binaries
* Cloning the SAAYN-Agent repository
* Building and validating the SAAYN CLI

## Procedure

1. Update the system package index.

   `student@ubuntu:~$` `sudo apt update && sudo apt upgrade -y`

2. Install required build tools.

   `student@ubuntu:~$` `sudo apt install git make build-essential -y`

3. Download a current Go release. The SAAYN repository requires a modern Go version. Use a recent stable release (1.24+).

   `student@ubuntu:~$` `wget https://go.dev/dl/go1.24.2.linux-amd64.tar.gz`

4. Extract Go to `/usr/local`.

   `student@ubuntu:~$` `sudo tar -C /usr/local -xzf go1.24.2.linux-amd64.tar.gz`

5. Configure your shell PATH.

   `student@ubuntu:~$` `echo 'export PATH=$PATH:/usr/local/go/bin:~/go/bin' >> ~/.bashrc`
   
   `student@ubuntu:~$` `source ~/.bashrc`

7. Verify the Go installation.

   `student@ubuntu:~$` `go version`

   ```
   go version go1.24.2 linux/amd64
   ```

8. Clone the SAAYN-Agent repository.

   `student@ubuntu:~$` `git clone https://github.com/sfeeser/saayn-agent.git`
   
   `student@ubuntu:~$` `cd saayn-agent`

9. Build the SAAYN binary.

   `student@ubuntu:~/saayn-agent$` `make build`

   Expected output:

   ```text
   🏗️  Building saayn...
   ✅ Build complete: ./saayn
   ```

10. Verify the SAAYN CLI runs.

   `student@ubuntu:~/saayn-agent$` `./saayn --help`

   Expected output:

   ```text
   A high-integrity, sovereign CLI to orchestrate the relationship between human intent and AI execution.

   Usage:
     saayn [command]

   Available Commands:
     edit
     help
     init
     reconcile
     undo
     verify

   Flags:
     -h, --help   help for saayn
   ```

## Conclusion

You have successfully prepared a SAAYN-compatible development environment. By installing a current Go toolchain and building the CLI from source, you now have a working SAAYN binary ready for initialization and inspection in the next lab.
